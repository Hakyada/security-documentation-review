# 🔐 Security Documentation Review

Automation tool developed during my M.Sc. thesis in Computer Science (IT Security focus).  
The script supports structured collection, analysis, and organization of security evaluation evidence.

---

## 📌 Overview

Security assessments often require collecting and reviewing large amounts of technical documentation such as:

- configuration files
- console outputs
- system logs
- test evidence
- security reports

Manual review is time-consuming and error-prone.

This script automates parts of the documentation review process by organizing and structuring security-related data for analysis and verification.

---

## 🎯 Purpose

The goal of this tool is to:

✔ automate evidence collection  
✔ improve reproducibility of security assessments  
✔ reduce manual documentation effort  
✔ support structured security evaluation workflows  
✔ assist compliance verification (e.g., ISO 27001 environments)

---

## ⚙️ What the Script Does

Depending on configuration, the script can:

- collect security-related system outputs
- organize files into structured directories
- process documentation for review
- support audit preparation
- standardize evaluation artifacts

This helps security engineers review system configurations more efficiently.

---

## 🧪 Use Case

This script is useful for:

- security assessments
- penetration testing documentation
- compliance verification
- audit preparation
- research environments
- IT security laboratories

Developed and tested in an academic security evaluation environment.
